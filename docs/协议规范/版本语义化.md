---

npm 包 version 应该遵循[版本语义化规范](https://semver.org/)

## why

1. 用来解决依赖地狱问题
2. [Why Use Semantic Versioning?](https://semver.org/#why-use-semantic-versioning)

## 知悉

1. 版本格式 MAJOR.MINOR.PATCH
2. 开发版本以 0.x.x 开头
3. pre-release 版本 x.x.x-x.x.x
   alpha->beta->rc->正式版本
